# COMP5523 项目测试报告

## 1. 测试目标

本次测试围绕“视障人士视觉辅助抓取系统”的核心功能展开，目标如下：

- 验证系统能否稳定识别指定目标物体。
- 验证系统能否稳定检测和追踪用户手部关键点。
- 验证系统能否根据手与目标之间的相对位置输出正确引导指令。
- 验证系统在实时视频处理中的响应速度与运行稳定性。
- 验证系统在模拟抓取任务中的辅助效果。
- 记录失败类型，为后续优化提供依据。

## 3. 测试方法

本次测试采用以下四类方法：

- 功能测试：验证检测、追踪、引导和抓取提示功能是否正确。
- 集成测试：验证各模块串联后能否稳定运行。
- 性能测试：记录时延、FPS 和长时间稳定性。
- 任务测试：模拟用户在系统引导下完成目标抓取。

## 8. 失败分析

### 8.1 主要失败原因

1. 目标遮挡
   当手接近目标时，部分帧中目标被手掌遮挡，导致目标检测结果不稳定。
2. 暗光环境
   在室内光线不足时，YOLO 和手部关键点检测都出现置信度下降。
3. 引导抖动
   当手部接近目标边缘时，`dx` 和 `dy` 的主导方向可能在相邻帧中快速切换，导致指令抖动。
4. 距离阈值误差
   当前 `distance < 50` 的抓取阈值是经验值，在不同摄像头角度和物距下可能不完全稳定。

### 8.2 改进建议

- 在引导逻辑中加入短时平滑或滞回阈值，减少指令抖动。
- 引入轻量目标跟踪机制，缓解目标短时遮挡问题。
- 在暗光条件下增加图像增强或补光措施。
- 对不同摄像头视角重新标定抓取阈值。
