# COMP5523 期末项目实现拆解：视障人士视觉辅助抓取系统

**项目主题**：Interactive Vision-Assisted Object Grasping for the Visually Impaired (基于交互式视觉辅助的视障人士目标抓取)
**核心目标**：开发一个实时音频引导系统，帮助视障人士在日常生活中安全、高效地拾取特定的目标物体（如鞋子、苹果、耳机等）。

---

## 整体实现思路 (Core Idea)
整个系统需要融合计算机视觉与语音交互技术，构成一个“感知-决策-交互”的闭环：
1. **感知（Tracking & Detection）**：同时追踪用户的手部位置和目标物体的位置。
2. **决策（Spatial Understanding）**：计算手与目标物体之间的相对空间距离和方位。
3. **交互（Real-time Audio Guidance）**：通过语音识别接收用户指令，并通过语音合成向用户提供实时移动指引（如“向右移动”、“向下移动”）。

---

## 详细步骤拆解与操作指南

### 步骤一：数据准备 (Data Preparation)
需要准备用于模型训练、微调或测试的数据，以确保系统在真实场景下的鲁棒性。

*   **实现思路**：搜集并整理目标物体、手部姿态以及抓取动作的数据集，为后续算法验证提供基础。
*   **具体操作步骤**：
    1.  **目标物体数据**：选择几个日常抓取目标（如杯子、手机、苹果）。可以直接使用开源数据集（如 COCO 或 Open Images），如果需要自定义物品，可录制视频并使用工具（如 LabelImg 或 Roboflow）进行 Bounding Box 标注。
    2.  **手部姿态数据**：无需从零训练，建议直接采用现有强大的预训练手部关键点数据集/模型（如 MediaPipe Hands、EgoHands 等）。
    3.  **抓取视频收集**：使用第一人称视角（Egocentric）设备录制正常的抓取动作，用于后期系统整体流程的离线测试和评估。

### 步骤二：算法设计 (Algorithm Design)
构建系统的核心处理逻辑，包括检测、空间计算和引导策略。

*   **实现思路**：模块化设计，将视觉检测和逻辑计算分离，定义清晰的音频引导触发条件。
*   **具体操作步骤**：
    1.  **手部检测与追踪**：集成 Google MediaPipe 或类似轻量级框架，实时提取手部的 2D/3D 关键点（重点关注掌心或食指指尖的坐标）。
    2.  **目标检测与追踪**：使用 YOLOv8 或 SSD 等实时目标检测模型，获取目标物体的中心点坐标。为防止物体短暂被手遮挡丢失，可加入简单的目标追踪算法（如 DeepSORT 或 ByteTrack）。
    3.  **空间关系理解**：
        *   计算手部中心点与目标物体中心点在画面中的相对向量（$\Delta x, \Delta y$）。
        *   （进阶）如果使用单目 RGB 摄像头，可通过目标检测框的面积变化（近大远小）粗略估算深度（$\Delta z$）；如果使用深度相机（如 RealSense），则直接获取 3D 距离。
    4.  **引导策略定义 (Guidance Strategy)**：设计一套状态机：
        *   *距离远*：给出大方向指引（如“向左移动”、“向前”）。
        *   *距离近*：给出微调指引（如“再靠近一点”、“张开手”）。
        *   *重合/极近*：提示“可以抓取”。

### 步骤三：系统实现 (System Implementation)
将各个算法模块整合为一个实时运行的端到端系统，并加入用户友好的语音交互接口。

*   **实现思路**：构建一个高效的视频处理流水线 (Pipeline)，保证低延迟，并实现语音的输入与输出。
*   **具体操作步骤**：
    1.  **视频流接入**：使用 Python 的 `cv2.VideoCapture` 读取摄像头实时画面，并按帧传入检测模型。
    2.  **语音指令识别 (ASR)**：集成 `SpeechRecognition` 或 `Whisper` 模型，让视障用户可以通过语音唤醒系统并指定目标（例如：“帮我找杯子”）。
    3.  **实时语音引导 (TTS)**：集成 `pyttsx3` 或 `gTTS` 库，将步骤二中计算出的引导策略（如 "move right"）转化为语音播放。
    4.  **多线程优化 (Multi-threading)**：**非常关键**。TTS 语音播放往往是阻塞式的，必须将视频帧的视觉处理（主线程）与语音播放（子线程）分开，避免系统卡顿导致引导滞后。

### 步骤四：评估与测试 (Evaluation)
对系统进行量化分析，这是 Project Report 中拿高分（Soundness & Writing）的关键。

*   **实现思路**：定义清晰的评价指标，进行多次实验，客观分析系统的优缺点。
*   **具体操作步骤**：
    1.  **指标定义**：记录系统的**成功率** (Success rate)、**引导准确度** (Guidance accuracy)、**抓取耗时** (Time-to-grasp) 以及**系统延迟** (Latency/FPS)。
    2.  **模拟测试**：邀请同学闭眼模拟视障人士，根据语音提示抓取不同位置的物体，记录上述数据。
    3.  **失败案例分析 (Failures)**：收集并分析失败情况，例如：手部完全遮挡目标导致检测丢失、光线暗淡导致识别失败等，并在报告中提出改进思路。

    2.  **Report 撰写**：结构清晰，语法准确。特别强调**本学期项目与上学期的区别**（上学期侧重于一般的场景理解和避障，本学期**更聚焦于特定任务的音频引导抓取**）。
